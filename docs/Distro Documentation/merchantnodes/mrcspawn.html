Controlling Merchant's Inventory

These are examples of the two types of elements in mrcspawn.cfg.
The first element defines the merchant type and the second element defines a group of items. Any item you wish a merchant to buy or sell must be in a group. The format of the elements and the properties within each element are pretty straight forward.

We will take a look at the MerchantType element first. The element key is the merchant's NPC template name as it appears in the npcdesc.cfg file. Each property contains a property name, either "Sells" or "Buys", followed by s string, the group of items from which the merchant sells or buys. As you can see with the Mage element, there are a number of magic related groups that can be sold or purchased by the NPC.

MerchantType  Mage
{
    Sells Reagents
    Sells Runes
    Sells Spellbook
    Sells Circle3spells
    Sells Circle2spells
    Sells Circle1spells
    Buys  reagents
    Buys  allscrolls
    Buys  runes
    Buys  wands
}

Now let's take a look at the ProductGroup element. As you can see in the example below you have the element type, ProductGroup, followed by a list of items in that group. The format for defining a ProductGroup property is:
"Item", ObjType number and the maximum amount of that item the NPC can have in stock. The comment after the item explains what item the ObjType is. The comments are not necessary but are highly recommended.

ProductGroup    Reagents
{
    Item    0xF7A  100 //black pearl
    Item    0xF7B  100 //blood moss
    Item    0xF84  100 //garlic
    Item    0xF85  100 //ginseng
    Item    0xF86  100 //mandrake root
    Item    0xF88  100 //nightshade
    Item    0xF8C  100 //sulphurous ash
    Item    0xF8D  100 //spider's silk
}

You can create additional product groups and add those to existing merchant's lists. If you create new merchants you can add them to mrcspawn.cfg by creating their own MerchantType element as well.


That is about it for understanding how to edit the mrcspawn.cfg file.
